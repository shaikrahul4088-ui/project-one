1. What is YAML & why use it?

English: YAML (“YAML Ain’t Markup Language”) is a human-readable format used by Kubernetes to define the desired state of resources (Deployments, Services, etc.). With YAML you describe what you want, and Kubernetes ensures the actual state matches.

3. Example YAML with explanation

English: Here’s a complete example (Deployment) with comments so you can follow.
Telugu (తెలుగు): క్రింద ఒక ఉదాహరణ YAML ఉంది. వ్యాఖ్యలతో కలిసి ఉన్నది కాబట్టి అన్నింటిని సులభంగా అర్థం చేసుకోవచ్చు.

apiVersion: apps/v1               # API version – apiVersion: apps/v1
kind: Deployment                  # Resource type – Deployment
metadata:
  name: demo-deployment           # Deployment పేరు
  labels:
    app: demo                      # లేబుల్
spec:
  replicas: 2                      # రెండు నకలు (pods) రన్ అవ్వాలి
  selector:
    matchLabels:
      app: demo                    # ఈ డిప్లాయ్‌మెంట్ ఏ నోడ్‌లను పంపించాలి అన్నది
  template:
    metadata:
      labels:
        app: demo                  # పాడ్స్ లోని లేబుల్
    spec:
      containers:
        - name: demo-container      # కంటైనర్ పేరు
          image: nginx:latest        # Docker ఇమేజ్
          ports:
            - containerPort: 80       # పోర్ట్
          livenessProbe:              # లైవ్‌నెస్ ప్రోబ్ (జీవిత సూచిక)
            httpGet:
              path: /
              port: 80
            initialDelaySeconds: 15
            periodSeconds: 10
          readinessProbe:             # రెడీనెస్ ప్రోబ్ (సర్వీస్ ఇవ్వడానికి సిద్ధంగా ఉన్నదా)
            httpGet:
              path: /
              port: 80
            initialDelaySeconds: 5
            periodSeconds: 5

4. Key fields explained

English / Telugu

replicas: How many pod instances you want. / మీరు ఎన్నో పాడ్‌లు ఉండాలని కోరుకుంటున్నారో.

selector + matchLabels: Helps Kubernetes know which pods are managed by this Deployment. / ఏ పాడ్‌లను ఈ డిప్లాయ్‌మెంట్ మేనేజ్ చేయాలో చెప్పేది.

template: The pod spec template. / పాడ్ ఎలా ఉండాలి అన్న టెంప్లేట్.

Under containers: image, ports etc. / కంటైనర్ ఇమేజ్, పోర్ట్ లాంటి వివరాలు.

livenessProbe: Checks if the container is alive; if fails then container restarts.

5. Applying the YAML

English:

Save the above text to a file, e.g., deployment.yaml.

Run:

kubectl apply -f deployment.yaml


To check status:

kubectl get deployments
kubectl get pods

6. Common pitfalls & tips

Ensure indentation is correct (YAML లో సపష్టం తప్పక).

Labels in selector.matchLabels must match the labels in template.metadata.labels. లేకపోతే pods సృష్టించబడవు.
